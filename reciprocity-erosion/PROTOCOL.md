# Reciprocity-erosion experiment — protocol

Companion experiment for *Beyond Memory: A Templated Substrate for Heterogeneous
Collaborative Knowledge Work with LLM Agents* (Saboia Moreira, Vardeman, Sweet).
It quantifies whether the `llm-wiki-memory-template`'s **verification gate**
preserves cross-reference (link) integrity in an LLM-maintained wiki as the wiki
is edited across many independent sessions — i.e. whether a discipline mechanism
the substrate ships is actually load-bearing over time.

## Research question

The wiki's reciprocity criterion says: if page A links to page B, page B should
reference A back, so a reader (or the knowledge graph) arriving at B can find A.
As a wiki accumulates edits across sessions, do these back-references erode, and
does running the verification gate prevent that erosion?

## Design

- **Unit:** one *chain* = one project instantiated from the template, edited over
  a sequence of **8 rounds**. Each round is a **fresh agent session with no memory
  of prior rounds** (it must rediscover wiki state from disk) — this reproduces the
  real driver of erosion, that later sessions do not remember earlier structure.
- **Conditions** (the only thing that differs between chains):
  - `gate-on` — each round is told to run the verification gate and ensure every
    new cross-reference is bidirectional (edit target pages as needed).
  - `gate-off` — each round is told NOT to run the gate and not to go back to add
    back-references; ingest the new material and move on. (Near-worst case.)
  - `natural` — each round is told to follow the wiki's normal documented Ingest
    procedure (CLAUDE.md + SCHEMA), no more and no less; the gate is neither
    emphasized nor suppressed. (The realistic middle; added after the first two.)
- **Replicates:** 3 chains per condition (separates trend from agent nondeterminism).
- **Rounds 1–4:** initial ingest of a fixed 4-document corpus (`corpus/`).
  **Rounds 5–8:** four follow-up notes (`rounds/round5..8.md`) that each introduce
  a new entity linking INTO already-existing pages — the reciprocity-breaking
  opportunity.

## Corpus

Fictional, self-contained (`corpus/` = rounds 1–4; `rounds/round5..8.md`).
Deliberately synthetic so the web of cross-references is fixed and identical for
every chain; the six core entities (Tidewatch, DriftNet, Kelp-Calibration,
Marren-Lab, Salinity-Drift-Finding, Bloom-Hypothesis) reference each other.

## Metric

`measure.py <wiki-dir>` emits, over content pages only (excluding Home/index/log/
SCHEMA/Edge-Types):
- `viol_typed_current_gate` — typed-edge links A→B where B does not reference A
  back by any means. (What the gate checks TODAY.)
- `viol_body_anymeans` — body links A→B where B does not reference A back.
  (What the proposed "any-means reciprocity" criterion would check.)

Measurement reads the wiki's **git history** (one commit per round, message
`roundN:`) via `measure-curve.sh` — it never trusts an agent's self-report.

## Provenance

- Template: `crcresearch/llm-wiki-memory-template` @ commit recorded in
  `template-snapshot.sha` at seed time (the runs in `results/` used `ad4b4c0`).
- Agents: run by Claude Code subagents; rounds orchestrated by
  `scripts/run-erosion-workflow.js` (chains parallel, rounds sequential).
- All runs were fully isolated in a scratch directory; no artifact from the
  template's own repo or wiki was read into or written by the chains.

## Reproducibility & honest limitations

- **Non-determinism is intrinsic.** Agents sample; "reproducible" here means the
  *trend and distribution* replicate, not identical counts. Report N and spread.
- **The experimenter is the system under study.** Data were generated by LLM
  agents and measured by an LLM-authored script; the script is deterministic and
  auditable (`measure.py`), but this coupling must be disclosed.
- **`gate-off` is near-worst-case** (reciprocity actively suppressed). The
  `natural` condition exists to estimate realistic erosion without that push.
- **Synthetic corpus, single model family, N=3.** Absolute counts do not transfer
  to predict a real wiki's totals; the *mechanism* (erosion accumulates without a
  mechanical check; the gate holds it near zero) is what the experiment supports.
- **`measure.py` is a lenient approximation** of the gate criterion (it accepts a
  back-reference by any means, including frontmatter). Validate against a gold
  standard before treating absolute violation counts as exact.

## Semantic scoring (does the mechanism produce *meaningful* edges?)

`semantic-metrics.py <wiki>` scores a wiki's link structure against
`ground-truth.json` — the relation set the corpus asserts. The corpus was fixed
before any run, so the ground truth is not results-dependent; each relation cites
the round file that asserts it and is auditable against it.

Metrics, all mechanical (no LLM judge, hence no LLM-judging-LLM circularity):
`recall` (ground-truth relations represented by any link), `typed_coverage`
(represented as a *frontmatter typed edge* — the form the KG pipeline consumes),
`spurious_rate` (canonical-to-canonical links whose pair the corpus never asserts
— the hallucination/padding proxy), `prose_backref_ratio` (back-references stated
in prose rather than only in a bare "See also" line). The scorer was validated by
injecting a spurious link (`spurious_rate` 0.0 → 0.5) before use.

## Phase 2 — pre-registered confirmatory battery

**This section was written and committed before Phase 2 was run; git history is
the audit trail.** Phase 1 (the exploratory n=3 batteries above, scored
post-hoc against a corpus-derived ground truth) suggested that per-write
enforcement suppresses KG-visible typed edges. Phase 2 tests that as a stated
hypothesis on fresh chains.

**Hypotheses (stated before data):**

- **H1** — `typed_coverage(nudge) < typed_coverage(natural)`: per-write mechanical
  enforcement reduces the share of true relations expressed as frontmatter typed
  edges. *Falsified if the replicate ranges overlap.*
- **H2** — `typed_coverage(periodic-lint) ≈ typed_coverage(natural)`: a periodic
  lint sweep does **not** reduce them, because the ingest agent never runs a check
  — only the sweep does. *Falsified if periodic-lint lands clearly below natural.*
- **H3** — `spurious_rate` does not increase under enforcement: the mechanism does
  not induce hallucinated or padded links. *Falsified if enforced conditions
  exceed natural's range.*
- **H4** — `recall(enforced) >= recall(natural)`: enforcement improves coverage of
  the relations the corpus actually asserts.

**Design:** conditions `natural`, `periodic-lint`, `nudge`; **n = 5** replicates
each (Phase 1's n=3 showed wide spread in `natural`); 8 rounds per chain, fresh
agent per round; all chains seeded from the **same commit** of the feature branch
(removing Phase 1's seed-source confound, where only some conditions carried the
checker). Primary metric: `typed_coverage` at final state. Secondary: `recall`,
`spurious_rate`, `prose_backref_ratio`.

**Analysis plan:** report medians and the **full replicate ranges** per condition.
No significance test is claimed at n=5 — overlap or separation of ranges is the
evidence, reported honestly either way. The scorer (`semantic-metrics.py`) and the
ground truth (`ground-truth.json`) are frozen at the commit that precedes the run.

**What it decides:** H2 holding supports merging the `/wiki-lint` mechanical check
(PR #84 on the template). H1 holding means a per-write hook would trade link
integrity for KG richness — a product decision to be made deliberately, and a
reason to design any hook to nudge *toward typed edges*, not merely toward
back-references.

## How to reproduce

```bash
# 1. seed chains from a template clone
TEMPLATE=~/src/llm-wiki-memory-template OUT=/tmp/exp \
  CONDITIONS="on off natural" REPLICATES=3 ./scripts/seed-chains.sh

# 2. run the 8 rounds per chain (Claude Code workflow; see run-erosion-workflow.js
#    — set `base` to $OUT and `chains` to the seeded ids, then run it)

# 3. measure from git history
OUT=/tmp/exp CONDITIONS="on off natural" REPLICATES=3 ROUNDS=8 \
  ./scripts/measure-curve.sh > results/battery.jsonl
```
